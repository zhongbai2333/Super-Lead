# 万物皆可绳 / Super Lead

[English](README.md)

**万物皆可绳** 是一个围绕独立 `Super Lead` 物品展开的 NeoForge 模组；它复用原版拴绳外观，但不会全局接管原版拴绳行为。

> 你有没有想过，拴绳除了拴猪，还能干点别的？

红石传导、能量输送、物品运输、流体流通，全都靠一根绳子。
操作简单，视觉直观，还带着那条原版拴绳的小垂线——
毕竟，连接本来就不需要那么复杂。

## 核心理念

Super Lead 保留原版拴绳最直观的体验：**两个点，一条看得见的连接线**。
它不是再做一套占方块空间的管道系统，也不是把连接关系藏起来的无线传输，
而是把拴绳变成一种轻量、直观、原版感十足的工程语言。

- 仅以 `Super Lead` 物品为基础；可由原版 `Lead` 合成转换。
- 锚点可以贴到方块任意面，不只是生物和栅栏。
- 保留原版拴绳的悬垂曲线，渲染为细方柱几何。
- 连接随世界保存，并实时同步给所有客户端。
- 所有升级和切换都是单次右键，不打开任何 GUI。
- 通过距离、容量和单根升级等级控制平衡。

## 拴绳种类

升级方式是**用对应材料右键已有的绳子（或副手的拴绳本体）**。每根绳子在同一时刻只有一种功能。

| 类型 | 转换材料 | 升级材料 | 颜色 | 用途 |
| --- | --- | --- | --- | --- |
| 普通 | — | — | 棕色 | 装饰 / 暂存 |
| 红石 | 红石块 | — | 红 / 深红 | 两端双向红石信号 |
| 能量 | 铁块 | 红石块 | 橙色 | FE 能量传输（按 tier 提升吞吐） |
| 物品 | 漏斗 | 箱子 | 钢蓝 | 物品传输（按 tier 提升单次数量） |
| 流体 | 炼药锅 | 桶 | 青绿 | 流体传输（按 tier 提升单次数量） |

升级时**客户端先确认目标**：你瞄准的那根绳子必须显示出虚影，服务端才会真的扣材料、做升级。这意味着用桶在炼药锅旁舀水时，只要你没瞄到绳子，桶不会被吞掉。

### 能量拴绳 (Energy Lead)

- 每根绳子独立保存功率与 tier。
- tier 0 即可在两端 FE 容器之间传输；每升一级**吞吐量翻倍**。
- 从 tier `T` 升到 `T+1` 共需 `1 << T` 个红石块（手中那个算一）。
- tier 上限可在配置文件中调整。
- 通电时绳子变亮，并随机喷发红石粒子；空闲时变暗。

### 物品拴绳 (Item Lead)

- 用**漏斗右键端点方块**可以**切换该端为提取端**，再点一次取消。
  当前提取端在视觉上会有一个**永远变粗的端头**作为标识。
- tier 0 单次传输 1 个物品；tier `t` 单次传输 `min(64, 1 << t)`，最多到 tier 6（满栈）。
- 每次升级需要 `1 << t` 个箱子（手中那个算一）。
- 每次成功传输都会沿提取方向走一段**蛇吞凸起动画**。
- **多绳路由**：多根物品绳挂在同一个栅栏结时，栅栏结自动作为分流口，按 DFS 加轮询遍历未访问过的下游绳子；目标拒收（容器满）会跳过而不破坏轮询节奏。

### 流体拴绳 (Fluid Lead)

- 切换 / 升级 / 动画规则与物品绳一致。
- 用**炼药锅右键端点方块**切换提取端。
- tier 0 单次传输 1000 mB（1 桶）；tier `t` 传输 `1000 mB × (1 << t)`，最多到 tier 4（16 桶/次）。
- 每次升级需要 `1 << t` 个桶（手中那个算一）。
- 兼容任何提供 `Capabilities.Fluid.BLOCK` 的方块。

## 视觉与交互

### 方块到方块连接

手持 Super Lead 右键方块面建立第一端锚点，再右键另一个方块即可建立连接。
锚点根据方块形状和点击面自动计算，常规方块、非完整方块、机器、栅栏和装饰方块都尽量自然落点。

### 物理绳子渲染

模组使用自定义绳子几何（细方柱），不再完全复用原版扁平 leash 条带。
客户端跑 Verlet 风格的绳子模拟：重力、阻尼、距离约束、方块 AABB 碰撞、绳子之间的互推。
**静止后会自动冻结**，避免不必要的物理消耗与抖动。

### 准星虚影 + 客户端确认

手持任一作用工具（剪刀、红石块、铁块、漏斗、箱子、炼药锅、桶）瞄准绳子时，绳子会出现一层颜色化的虚影。这层虚影既是视觉提示，也是触发条件：

- 客户端先用准星确认到底瞄准了哪根绳子；
- 然后向服务端发送 `UseConnectionAction` 包；
- 服务端做距离与持物校验后，**只对客户端确认的那根绳子**执行升级或切割。

最直接的好处：**没看到虚影 = 不会触发**。在炼药锅旁用桶舀水、用漏斗放置等原版操作不会被误吞。

### 剪断连接

剪刀右键世界中的任一根绳子即可剪断，自动掉落对应数量的 Super Lead 物品。
剪刀右键栅栏结会一次切断结上的所有绳子。

### 提示 (Tooltip)

手持剪刀瞄准绳子时，准星附近显示该绳的种类、tier 倍率（如适用）、是否通电等信息。

## 持久化与网络

- 连接以维度为单位保存到 `SavedData`，世界重载、服务器重启都会保留。
- 端点方块被破坏、变成空气、超距，连接会被自动清理。
- 栅栏结在世界加载时按需重建。
- 区块级同步使用 `SyncRopeChunk`，登录/换维度用 `ClearRopeCache` 清旧缓存；物品/流体每次成功传输还会附送一个微型 `ItemPulse` 包驱动凸起动画。

## 平衡设计

Super Lead 不打算替代专业管道或物流模组。它强调灵活、可视、原版感：

- 单根绳子最大长度 12 格；
- 每根绳子只承载一种功能；
- 全部连接都在世界中可见，没有隐藏网络；
- 升级材料按 `2^tier` 指数递增；
- 两端必须具备对应能力；
- 无效或过远的连接每 tick 自动剔除。

## 开发路线

| 里程碑 | 状态 | 内容 |
| --- | --- | --- |
| 1. 可连接原型 | 已完成 | 方块面锚点、连接持久化、世界中可见绳子。 |
| 2. 功能拴绳 | 已完成 | 普通 / 红石 / 能量 / 物品 / 流体 五种。 |
| 3. 传输系统 | 已完成 | 分级 FE / 物品 / 流体传输，提取端切换，栅栏结分流，逐次脉冲动画。 |
| 4. 展示打磨 | 进行中 | 提示文本、客户端校验、音效、粒子、展示截图。 |

## 为什么不是管道？

- 比管道更轻量，不占完整方块空间。
- 比无线传输更直观，连接关系直接出现在世界里。
- 比全新系统更原版，玩家拿到拴绳就知道该怎么试。

一根普通的绳子，连接不普通的世界。

## 环境需求

- Minecraft：`26.1.2`
- NeoForge：`26.1.2.43-beta` 或兼容的 `26.x`
- Java：`25`

## 构建

基于 Gradle 的 NeoForge 项目。在仓库根目录：

```bash
./gradlew build
```

成品 jar 位于 `build/libs/`。
